About this App

I built this app to keep track of my climbing progress. Although there are already many tracking apps on the market, mine collects much more data compared to other apps, so it made sense for me to build a custom solution.

On a high-level, this app executes SQL queries on data stored in a Heroku Postgres database, then uses the returned results to generate visualizations using Plotly.

On a more detailed-level, here's how I made the app run more smoothly:
  1. When opening a dashboard page for the first time after bringing the app out of a sleeping website state, the app opens a connection to a Heroku Postgres database and attempts to retrieve the data. If the connection is dropped or no data is returned, the app instead retrieves the data from the CSV files saved on the disk. Finally, the app generates the figures and saves it to Heroku's ephemeral disk, then displays the figures on the webpage.
  2. When refreshing a dashboard page while the app is still active, the app reads in the figure directly from Heroku's ephemeral disk instead of regenerating it.
